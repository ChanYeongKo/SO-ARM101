# Pick and Dump: SO-ARM101 쓰레기통 비우기 시스템

SO-ARM101 5DOF 로봇팔과 Intel RealSense D405 카메라를 사용하여, **쓰레기통을 자동으로 파지 → 비우기 → 제자리에 돌려놓는** 시스템입니다.

---

### 1. 물체 검출 (YOLO + Depth Camera)

```
┌─────────────────────────────────────────────────────────┐
│                물체 검출 파이프라인                        │

│  │ RGB    │  │ Depth  │                                 │
│  │ 640x480│  │ 640x480│                                 │
│  └───┬────┘  └───┬────┘                                 │
│      │           │                                      │
│      ▼           │                                      │
│  ┌────────┐      │                                      │
│  │ YOLOv8 │      │  커스텀 학습 모델 (best.pt)           │
│  │ 추론   │      │  confidence > 0.5                    │
│  └───┬────┘      │                                      │
│      │           │                                      │
│      ▼           ▼                                      │
│  ┌──────────────────┐                                   │
│  │ bbox (x1,y1,x2,y2)                                  │

│  │                                      │               │

- **YOLO 모델**: 커스텀 학습된 쓰레기통 검출 모델
- **Depth 처리**: 바운딩박스 중앙 80% 영역의 유효 depth 중앙값 사용 (엣지/노이즈 제거)
- **적응형 파지 높이**: 물체 높이의 70% 지점에서 파지 (높이 2~10cm 범위 제한)

### 2. 좌표 변환 (Camera → Base Frame)

Eye-in-hand 구성에서 카메라 좌표를 로봇 베이스 좌표로 변환하는 2단계 과정입니다.

```
┌─────────────────────────────────────────────────────────┐
│            좌표 변환 파이프라인 (Eye-in-Hand)             │
│                                                         │
│  카메라 프레임                 그리퍼 프레임               │
│  ┌──────────┐                ┌──────────┐               │
│  │ P_cam    │  T_cam→gripper │ P_gripper│               │
│  │ (Xc,Yc, │ ──────────────►│ (Xg,Yg, │               │
│  │  Zc)    │  (Hand-Eye     │  Zg)    │               │
│  └──────────┘   캘리브 결과)  └────┬─────┘               │
│                                    │                     │
│                    FK(q_current)   │                     │
│                    T_base→gripper  │                     │
│                                    ▼                     │
│                              ┌──────────┐               │
│   베이스 프레임               │ P_base   │               │
│                              │ (Xb,Yb, │               │
│                              │  Zb)    │               │
│                              └──────────┘               │
│                                                         │
│  수식:                                                   │
│  P_gripper = T_cam→gripper × P_cam                      │
│  P_base    = T_base→gripper(FK) × P_gripper             │
│                                                         │
│  ※ T_cam→gripper: Hand-Eye 캘리브레이션 고정값            │
│  ※ T_base→gripper: FK로 현재 관절각에서 실시간 계산       │
└─────────────────────────────────────────────────────────┘
```

### 3. Forward Kinematics (FK)

URDF 모델과 현재 관절각으로 그리퍼의 3D 위치/자세를 계산합니다.

```
┌─────────────────────────────────────────────────────────┐
│                  Forward Kinematics                      │
│                                                         │
│  입력: q = [θ1, θ2, θ3, θ4, θ5] (5개 관절각, deg)       │
│                                                         │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐        │
│  │ base_link  │─►│ shoulder   │─►│ upper_arm  │        │
│  │            │  │ θ1 (pan)   │  │ θ2 (lift)  │        │
│  └────────────┘  └────────────┘  └─────┬──────┘        │
│                                        │                │
│  ┌────────────┐  ┌────────────┐  ┌─────┴──────┐        │
│  │ gripper    │◄─│ wrist      │◄─│ lower_arm  │        │
│  │ (TCP)     │  │ θ4(flex)   │  │ θ3 (elbow) │        │
│  │           │  │ θ5(roll)   │  │            │        │
│  └────────────┘  └────────────┘  └────────────┘        │
│                                                         │
│  출력: T_base→gripper (4×4 변환행렬)                     │
│  ┌                              ┐                       │
│  │  R(3×3)  │  t(3×1)          │                       │
│  │  ────────┼──────────        │                       │
│  │  0  0  0 │  1               │                       │
│  └                              ┘                       │
│  • R: 그리퍼 자세 (회전행렬)                              │
│  • t: 그리퍼 위치 (x, y, z) [m]                         │
│                                                         │
│  사용처:                                                 │
│  • 좌표 변환 (cam→base)의 2단계                          │
│  • Hand-Eye 캘리브레이션 데이터 수집                      │
└─────────────────────────────────────────────────────────┘
```

### 4. Inverse Kinematics (IK)

목표 위치(x, y, z)에서 필요한 관절각을 계산합니다.

```
┌─────────────────────────────────────────────────────────┐
│                  Inverse Kinematics                      │
│                                                         │
│  입력: target_pos = [x, y, z] (베이스 프레임, meters)    │
│                                                         │
│  ┌──────────────────────────────────────────────┐       │
│  │           다중 후보 IK 풀이                    │       │
│  │                                              │       │
│  │  1. Top-Down 후보 12개 (사전 정의 초기값)     │       │
│  │     → 그리퍼가 위에서 아래로 내려오는 자세 유도 │       │
│  │     → orientation_hint: Y축 -90° 회전         │       │
│  │     → ori_weight: 0.01 (위치 우선)            │       │
│  │                                              │       │
│  │  2. 현재 자세 기반                            │       │
│  │     → 현재 관절각에서 출발                     │       │
│  │                                              │       │
│  │  3. 홈 자세 기반                              │       │
│  │     → 모든 관절 0°에서 출발                    │       │
│  │                                              │       │
│  │  각 후보에 대해:                               │       │
│  │  • 위치 오차 (mm) 계산                        │       │
│  │  • 그리퍼 하향도 (downward_score) 계산         │       │
│  │  • 오차 < 25mm인 해만 수집                    │       │
│  └───────────────────────┬──────────────────────┘       │
│                          │                              │
│                          ▼                              │
│  ┌──────────────────────────────────────────────┐       │
│  │           최적 해 선택                         │       │
│  │                                              │       │
│  │  정렬 기준:                                   │       │
│  │  1순위: downward_score 높은 것 (아래 향함)     │       │
│  │  2순위: 위치 오차 작은 것                      │       │
│  │                                              │       │
│  │  → 위에서 내려오면서 가장 정확한 해 선택       │       │
│  └──────────────────────────────────────────────┘       │
│                                                         │
│  출력: q = [θ1, θ2, θ3, θ4, θ5] (deg)                  │
│                                                         │
│  5DOF 제약:                                              │
│  • 임의의 6DOF 자세 불가                                 │
│  • 위치(3DOF) 우선, 자세는 "위에서 내려오는" 방향 유도    │
│  • pitch_sum 보정으로 그리퍼 각도 별도 제어               │
└─────────────────────────────────────────────────────────┘
```

### 5. Pitch Sum Conservation (그리퍼 각도 보존)

5DOF 로봇의 핵심 보정 알고리즘입니다. IK가 위치만 풀어주므로, 그리퍼의 절대 피치 각도는 별도로 보정해야 합니다.

```
┌─────────────────────────────────────────────────────────┐
│              Pitch Sum Conservation                      │
│                                                         │
│  원리:                                                   │
│  그리퍼의 절대 피치 각도 =                                │
│    shoulder_lift + elbow_flex + wrist_flex               │
│                                                         │
│  이 합(pitch_sum)을 일정하게 유지하면,                    │
│  팔이 어떤 자세를 취하든 그리퍼는 같은 각도로              │
│  물체를 향합니다.                                         │
│                                                         │
│  ┌─────────────────────────────────────────────┐        │
│  │  1. 파지 미리보기 IK → pitch_sum 결정        │        │
│  │                                             │        │
│  │  target_pitch_sum =                         │        │
│  │    preview[SL] + preview[EF] + preview[WF]  │        │
│  │    + WRIST_FLEX_EXTRA - 10                  │        │
│  └──────────────────┬──────────────────────────┘        │
│                     │                                    │
│                     ▼                                    │
│  ┌─────────────────────────────────────────────┐        │
│  │  2. 매 이동 단계마다 wrist_flex 재계산       │        │
│  │                                             │        │
│  │  wrist_flex = target_pitch_sum              │        │
│  │              - shoulder_lift                 │        │
│  │              - elbow_flex                    │        │
│  │                                             │        │
│  │  → IK가 SL, EF를 바꿔도 WF 보정으로         │        │
│  │    그리퍼 각도 유지                          │        │
│  └─────────────────────────────────────────────┘        │
│                                                         │
│  예시: pitch_sum = 19°                                   │
│                                                         │
│  팔 세우기:  SL=90  EF=0   → WF = 19-90-0  = -71°      │
│  목표 위:    SL=44  EF=58  → WF = 19-44-58 = -83°      │
│  하강:       SL=50  EF=40  → WF = 19-50-40 = -71°      │
│                                                         │
│  → 모든 단계에서 그리퍼 절대 각도 = 19° (일정!)          │
└─────────────────────────────────────────────────────────┘
```

### 6. Hand-Eye Calibration (캘리브레이션)

│  ┌─────────────────────────────────────────────┐        │

│  AX = XB 문제:                                           │

│  B: 보드 관측 변화 (카메라)                               │
│  X: T_cam→gripper (구하려는 값)                           │
└─────────────────────────────────────────────────────────┘
```

## 사용법

```bash
cd ~/lerobot2/project

```

1. 로봇이 SAFE_POSE로 자동 이동
